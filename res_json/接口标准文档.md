# 粉笔题库接口文档

本文档根据 `res_json/接口.md` 中的 Fiddler 抓包记录整理。原始文件保留完整响应样本和当时的 Cookie；本文件只描述接口契约，不保存明文认证信息。

## 监听与复放结论

- 本机 `127.0.0.1:8888` 当前存在监听服务，进程为 `Fiddler.exe`。
- 使用文档中的三个 URL 和原始 Cookie 直接请求，以及显式走 `http://127.0.0.1:8888` 代理请求，三个接口当前均返回 `HTTP 453`。
- 结论：当前环境可以连接 Fiddler 代理，但未能获取三个接口的新成功响应。更可能的原因是原始 Cookie 已过期、服务端风控校验不满足，或浏览器登录态/请求指纹与复放请求不一致。
- 若要实时监听成功响应，需要在已登录的浏览器中走 Fiddler 代理，刷新对应页面后在 Fiddler 中过滤下方三个 URL path。

## 公共信息

### 认证方式

接口依赖浏览器登录态，原始抓包使用 `Cookie` 请求头认证。

请求时应从当前登录浏览器会话中获取 Cookie，不建议把 Cookie 写入仓库文档。

### 公共请求头

| Header | 示例 | 说明 |
| --- | --- | --- |
| `Cookie` | `<从浏览器登录态获取>` | 必填，用户登录态 |
| `Accept` | `application/json, text/plain, */*` | 建议 |
| `User-Agent` | 浏览器 UA | 建议保持与浏览器一致 |
| `Referer` | `https://www.fenbi.com/` 或实际页面地址 | 建议保持与触发页面一致 |

### 公共查询参数

| 参数 | 示例 | 说明 |
| --- | --- | --- |
| `app` | `web` | 客户端类型 |
| `kav` | `128` | Web 版本相关参数，按抓包保留 |
| `av` | `128` | Web 版本相关参数，按抓包保留 |
| `hav` | `128` | Web 版本相关参数，按抓包保留 |
| `version` | `3.0.0.0` | 版本号 |
| `deviceId` | 空字符串 | 设备 ID，原始抓包为空 |
| `gav` | `2` | 网关/版本相关参数，按抓包保留 |
| `apcId` | `0` | 按抓包保留 |

### 通用响应约定

部分接口返回对象包装：

```json
{
  "code": 1,
  "msg": "SUCCESS",
  "data": {}
}
```

`code = 1` 表示业务成功。第三个错题树接口直接返回数组，不使用上述包装对象。

## 接口一：获取练习历史

### 基本信息

| 项目 | 内容 |
| --- | --- |
| 名称 | 获取练习历史 |
| Method | `GET` |
| Host | `https://tiku.fenbi.com` |
| Path | `/combine/exercise/getExerciseBriefHistory` |
| 原始状态 | `200 OK` |
| 当前复放 | `HTTP 453` |

### 请求参数

| 参数 | 类型 | 必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| `noCacheTag` | number/string | 否 | `940` | 防缓存参数 |
| `categoryId` | number | 是 | `3` | 题库分类 ID |
| `limit` | number | 否 | `15` | 每页数量 |
| `cursor` | string | 否 | 空字符串 | 分页游标，第一页为空 |
| `routecs` | string | 是 | `xingce` | 科目路由 |
| `app` | string | 是 | `web` | 公共参数 |
| `kav` | number/string | 是 | `128` | 公共参数 |
| `av` | number/string | 是 | `128` | 公共参数 |
| `hav` | number/string | 是 | `128` | 公共参数 |
| `version` | string | 是 | `3.0.0.0` | 公共参数 |
| `deviceId` | string | 否 | 空字符串 | 公共参数 |
| `gav` | number/string | 是 | `2` | 公共参数 |
| `apcId` | number/string | 是 | `0` | 公共参数 |

### 请求示例

```http
GET /combine/exercise/getExerciseBriefHistory?noCacheTag=940&categoryId=3&limit=15&cursor=&routecs=xingce&app=web&kav=128&av=128&hav=128&version=3.0.0.0&deviceId=&gav=2&apcId=0 HTTP/1.1
Host: tiku.fenbi.com
Cookie: <browser-cookie>
Accept: application/json, text/plain, */*
```

### 响应字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | number | 业务状态码，成功为 `1` |
| `msg` | string | 业务消息，样本为 `SUCCESS` |
| `data.cursor` | string | 下一页游标，例如 `3949698356_14` |
| `data.historyItems` | array | 练习历史列表 |
| `data.historyItems[].exerciseId` | number | 练习 ID |
| `data.historyItems[].exerciseKey` | string | 练习 key |
| `data.historyItems[].sheetName` | string | 练习名称 |
| `data.historyItems[].sheetType` | number | 练习类型 |
| `data.historyItems[].difficulty` | number | 难度 |
| `data.historyItems[].status` | number | 状态，样本中 `0` 为未完成，`1` 为已完成 |
| `data.historyItems[].updatedTime` | number | 更新时间，毫秒时间戳 |
| `data.historyItems[].questionCount` | number | 题目数 |
| `data.historyItems[].correctCount` | number | 答对数 |
| `data.historyItems[].score` | number | 得分 |
| `data.historyItems[].fullScore` | number | 满分 |
| `data.historyItems[].hasVideo` | number | 是否有视频 |
| `data.historyItems[].miniTitle` | string/null | 小标题 |

## Fiddler 监听建议

1. 确认浏览器代理指向 `127.0.0.1:8888`。
2. 确认 Fiddler 已开启 HTTPS 解密，否则只能看到 CONNECT 隧道，无法查看 JSON 响应体。
3. 登录粉笔 Web 页面后分别访问：
   - 练习历史页：触发 `/combine/exercise/getExerciseBriefHistory`
   - 用户报告页：触发 `/api/v3/user/analysis`
   - 我的错题页：触发 `/api/xingce/errors/keypoint-tree`
4. 在 Fiddler 的 `Filters` 或搜索中使用以下 path 过滤：
   - `getExerciseBriefHistory`
   - `/api/v3/user/analysis`
   - `/errors/keypoint-tree`
5. 成功监听时应看到 HTTP 状态码 `200`，响应体为 JSON，且业务成功响应中 `code = 1`。

## 当前复测命令

PowerShell 复测可使用如下模板。Cookie 需要替换为当前浏览器登录态。

```powershell
$headers = @{
  Cookie = '<browser-cookie>'
  Accept = 'application/json, text/plain, */*'
  'User-Agent' = '<browser-user-agent>'
  Referer = 'https://www.fenbi.com/'
}

Invoke-WebRequest `
  -Uri 'https://tiku.fenbi.com/combine/exercise/getExerciseBriefHistory?noCacheTag=940&categoryId=3&limit=15&cursor=&routecs=xingce&app=web&kav=128&av=128&hav=128&version=3.0.0.0&deviceId=&gav=2&apcId=0' `
  -Headers $headers `
  -Proxy 'http://127.0.0.1:8888'
```
